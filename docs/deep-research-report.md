## Executive summary

A high-quality VHF/DSC simulator benefits most from (a) authoritative **channel/procedure references**, (b) **safe-to-reuse UI iconography and radio-console components**, and (c) **radio-style audio textures** (static, squelch, beeps) that can be mixed with learner speech and verified via an online transcription model.

- **Public-domain U.S. government references** for channel usage and voice procedures (e.g., FCC charts; USCG NAVCEN MAYDAY procedure). U.S. federal works are generally **not copyrightable in the U.S.** under 17 U.S.C. ¬І105. оИАciteоИВturn1search3оИВturn23view0оИВturn10view1оИБ

## Implementation checklist for audio assets with an AI transcription pipeline

This checklist assumes your simulator is вАЬonline-verifiedвАЭ: learners speak into the mic, you transcribe server-side, and compare to expected procedures.

### Audio capture and preprocessing

- Capture mic audio with a consistent pipeline (Web Audio API is the typical browser foundation for capture/routing/mixing). оИАciteоИВturn4search13оИБ
- Maintain **two audio tracks**:
  - **Clean track** (minimal/no added static): used for transcription and scoring.
  - **Presentation track** (with static/squelch beeps): used for userдљУй™М/realism playback.
- Keep SFX levels low and consider ducking static under speech (or disable static during scoring) so the transcription model is evaluating the learnerвАЩs radio phraseology, not your noise bed.

### Encoding formats and sample rates for transcription

Two common patterns (choose one and standardize):

- **File-based transcription (batch / per-attempt)**: Use one of the API-supported formats (e.g., `wav`, `mp3`, `m4a`, `webm`). OpenAIвАЩs speech-to-text guide lists supported upload types and a 25 MB limit. оИАciteоИВturn24search0оИБ
- **Streaming / real-time transcription**: OpenAIвАЩs realtime transcription guide documents supported input formats including `audio/pcm` **24 kHz mono PCM** and optional built-in noise reduction modes. оИАciteоИВturn24search8оИБ
  - If using realtime transcription sessions, OpenAIвАЩs reference indicates `pcm16` expectations: **16-bit PCM, 24 kHz, mono, little-endian**. оИАciteоИВturn24search11оИБ

### Noise handling (important for вАЬradio realismвАЭ + вАЬfair scoringвАЭ)

- If using OpenAI realtime transcription, consider `audio.input.noise_reduction` with `near_field` for headset/mic usage or `far_field` for speakerphone-like setups. оИАciteоИВturn24search8оИБ
- Build difficulty levels by adjusting **noise mix**, not by degrading the learnerвАЩs own mic track:
  - Level 1: no static added
  - Level 2: low static bed
  - Level 3: static + occasional dropouts (use CC0 вАЬStatic.wavвАЭ with breakup) оИАciteоИВturn21view0оИБ

### Transcript scoring approach (robust against minor wording variance)

- Prefer **slot-based scoring** for structured calls:
  - Required tokens (e.g., вАЬMAYDAYвАЭ x3; вАЬTHIS ISвАЭ; ship name; position; nature of distress; assistance; persons onboard). оИАciteоИВturn10view1оИБ
- Allow tolerance for punctuation and small filler words; focus on **presence and order** of critical fields.
